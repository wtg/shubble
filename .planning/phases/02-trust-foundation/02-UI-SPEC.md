## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline badge padding vertical, icon gaps |
| sm | 8px | Badge horizontal padding, compact element spacing, secondary timeline item gap |
| lg | 16px | Timeline content padding, default element spacing, container padding |
| 2xl | 24px | Section padding, timeline item margins |

Exceptions: Existing `Schedule.css` uses non-8pt values (5px, 6px, 10px, 12px, 14px, 15px, 20px, 25px, 30px). Existing values are left as-is. All NEW spacing introduced by this phase uses only the tokens above.

## Typography

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Body / Missing data message | 14px | 400 | 1.5 | System stack |
| ETA time (primary) | 18px | 700 | 1.3 | System stack |
| Source badge (LIVE/SCHED) / Deviation badge | 12px | 700 | 1.2 | System stack |
| Page heading | 28px | 400 | 1.2 | Karla |

**Declared sizes (4):** 12px, 14px, 18px, 28px
**Declared weights (2):** 400, 700

Design rationale:
- 13px missing data message merged to 14px body; italic style provides sufficient visual distinction.
- 10px badge merged to 12px for readability and scale compliance.
- Weights collapsed to 400 (normal) and 700 (bold). ETA primary promoted to 700 for emphasis. Former 500/600 intermediates dropped.

Source: Existing `Schedule.css` values. Badge sizes from RESEARCH.md code examples (Claude's discretion per CONTEXT.md).

## Copywriting Contract

| Element | Copy | Condition |
|---------|------|-----------|
| LIVE badge | `LIVE` | ETA from GPS data (live or lastArrival) |
| SCHED badge | `SCHED` | ETA from schedule fallback |
| Early deviation | `-{N} min early` | Live ETA > 2 min before scheduled time |
| Late deviation | `+{N} min late` | Live ETA > 2 min after scheduled time |
| No service (off hours) | `No shuttle in service` | Current time is after last departure or no schedule for today |
| Pre-service | `Service starts at {X:XX AM}` | Current time is before first departure of the day |
| No service (no schedule) | `No shuttle in service` | No entries in aggregated_schedule for this day/route |

### States Not Included (Intentional)

| State | Why Excluded | What Shows Instead |
|-------|-------------|-------------------|
| "En route to first stop" | D-10: scheduled fallback covers this case | Scheduled time + SCHED badge |
| On-time deviation | D-11: within 2-min dead zone | No deviation badge shown |
| Very late (red tier) | D-13: keep simple with 2 tiers | Orange (+N min late) for all late > 2 min |

### Error/Edge Cases

| Scenario | Display |
|----------|---------|
| Live ETA exists but no scheduled time for comparison | Show live ETA + LIVE badge, no deviation badge |
| lastArrival (shuttle already passed stop) | Show lastArrival time + LIVE badge, no deviation badge |
| Schedule data missing for current day | `No shuttle in service` |
| Deviation exactly +/- 2 minutes | No badge (within dead zone, inclusive) |

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Badge tap/click | No interaction (badges are informational only) |
| Deviation badge visibility | Appears/disappears on 30-second re-render tick |
| Missing data message transition | No animation (Phase 4 handles transitions) |
| Source badge on expanded timeline | Visible on every stop row when secondary timeline is expanded |

## Accessibility Contract

| Requirement | Implementation | Source |
|-------------|---------------|--------|
| Text labels not color alone | LIVE/SCHED text badges accompany colored ETA text | D-04, Success criterion #4 |
| Deviation text labels | "+3 min late" / "-1 min early" text, not just orange/blue color | D-04 |
| Color contrast (LIVE badge) | `#578FCA` on white = 3.5:1 (supplementary, text provides meaning) | WCAG AA for non-text |
| Color contrast (SCHED badge) | `#95a5a6` on white = 2.7:1 (supplementary, text provides meaning) | WCAG AA for non-text |
| Missing data messages | Italic text at 14px with `#95a5a6` color; semantic meaning conveyed by text content | D-08 |
