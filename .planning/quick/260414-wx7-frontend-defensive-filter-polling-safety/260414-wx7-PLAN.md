<objective>
Harden the Schedule page against stale/cross-route trips in browser state by adding (1) a defensive filter that drops any trip whose stop_etas keys don't belong to the selected route, and (2) a safety-net background poll in useTrips that runs regardless of SSE health so a silently-stalled EventSource can't freeze the UI beyond ~10s.

Purpose: Users reported seeing a WEST-route schedule row expand to NORTH stops (Colonie/Georgian/Bryckwyck) for vehicle 001. Live `/api/trips` polling shows ZERO cross-route trips right now, so the bad data is cached in React state from an earlier moment. Hard refresh clears it. Root cause is documented exhaustively in task_context: SSE keepalives keep the EventSource "open" but if Redis pub/sub stalls, no new `data:` payload arrives and `setTrips` never fires again. This plan adds two layered defenses.

Output: Two atomic commits, both frontend-only, both narrowly scoped. No backend changes. No new runtime dependencies.
</objective>

```typescript
export function useTrips(enabled = true, pollInterval = 5000) {
  const [trips, setTrips] = useState<Trip[]>([]);

  useEffect(() => {
    if (!enabled) return;

    const controller = new AbortController();
    let cancelled = false;
    let pollTimer: ReturnType<typeof setInterval> | null = null;
    let es: EventSource | null = null;

    const applyTrips = (data: unknown) => {
      if (Array.isArray(data) && !cancelled) setTrips(data as Trip[]);
    };

    const fetchTrips = async () => { /* ... */ };

    const startPolling = () => {
      if (pollTimer !== null || cancelled) return;
      void fetchTrips();
      pollTimer = setInterval(() => { void fetchTrips(); }, pollInterval);
    };

    const stopPolling = () => {
      if (pollTimer !== null) { clearInterval(pollTimer); pollTimer = null; }
    };

    void fetchTrips();  // initial

    if (typeof EventSource !== 'undefined') {
      try {
        es = new EventSource(`${config.apiBaseUrl}/api/trips/stream`);
        es.onopen = () => { stopPolling(); };   // stops FAST poll on SSE open
        es.onmessage = ...;
        es.onerror = () => {
          if (es?.readyState === EventSource.CLOSED) { startPolling(); }
        };
      } catch { startPolling(); }
    } else {
      startPolling();
    }

    return () => {
      cancelled = true;
      controller.abort();
      stopPolling();
      if (es) { es.close(); es = null; }
    };
  }, [enabled, pollInterval]);

  return trips;
}
```

The SAFETY poll is independent of `startPolling`/`stopPolling` — its timer is never cleared by SSE `onopen`. It keeps running for the lifetime of the effect and is cleaned up on unmount.
